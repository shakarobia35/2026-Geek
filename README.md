### 🤔有关学习、思考小Tips

1. 学会如何使用 AI 大模型解决问题，学会**适度使用**而不是过度依赖

   > AI 是工具，不是拐杖，
   >
   > AI 可以帮助你加速学习，但不能替你完成考核
   >
   > 学会用它解决问题，但**别让它替你思考**

   AI 可用于：查概念、查资料、排查报错、辅助理解代码等，
   但需要自己判断：**我是否真的理解了？**

   建议在文档或编后感中简单说一下：

   * 哪些地方用到了 AI
   * 哪些内容是你自己完成的
   * 做完之后你大概理解了什么

2. 遇到问题，先搜索，先自己想；
   想清楚了问题是什么，再去问人——带着具体问题问，比带着模糊困惑问，收获多得多
   推荐阅读[提问的智慧](https://github.com/ryanhanwu/How-To-Ask-Questions-The-Smart-Way/blob/master/README-zh_CN.md)

3. 强烈建议使用 Google 搜索引擎搜索所需内容，可以尝试思考一下为什么我们推荐 Google 

   > 如果发现用不了Google，可以进一步思考原因以及解决方案

4. 好奇之心，改变之力

   > 或许你现在还不知道自己向往什么、感兴趣什么——但好奇之心本身就是力量，往前走吧。

## 1️⃣阶段一 【Geek 的出厂设置 - `GitHub` 入门与 Markdown 学习】

### 阶段时长：`1.5` 天

### 🛒学习内容

1. 初步了解 `GitHub` 

2. 学习使用 Markdown

   > Markdown 可能是未来 Geek 们接触最多的工具，当你真正爱上它的时候，电脑中的 Word 似乎再也没怎么打开过......

### 📑任务内容

**❗❗❗注意：考核期所有笔记/文档须为 `.md` 格式，当前与此后的所有任务内容均上传至你在 `GitHub`创建的名为 Tasks 的仓库**

1. 用上你学的 Markdown 语法，组织一份整洁规范的文档，跟我们说说：
   - 你觉得什么是Geek，什么又是 Geek 精神？
   - GitHub 是干什么的？
     Pull Request、Fork 这些词你是怎么理解的，又是从哪里学到的？
   - 有学长当年光是打开 GitHub 网站就花了半天，你有过类似经历吗？
     不管有没有，说说你的判断。
   - 这份文档凭什么称得上整洁规范？你遵循了哪些规范？

2. 创建 Profile README 仓库，美化你的 GitHub 主页
   - **参考至少 2 个优秀的 GitHub Profile README(并给出对应的参考链接🔗)**      
   - 记录📝在你的学习笔记中记录你参考了哪些案例，学到了什么技巧

3. **[选做-进阶]** 了解 Markdown 的底层原理，探索一下你的编辑器还有什么奇妙或者方便的功能，写成文档分享给大家和未来的学弟学妹吧。

### 📑这阶段后，你的仓库里应该有

> **命名仅推荐**，大家可以慢慢找到适合自己的整理习惯以及命名习惯

* **个人 Profile README 仓库**

* **GitHub 学习笔记**

* **Markdown 学习笔记**

* **Markdown 练习文档**

* **[可选] Markdown 进阶笔记**

* **[可选] 问题解决记录**

  > 除以上内容外，我们也很希望看见你的其他思考——无论大或小，深或浅，无论有关什么

## 3️⃣阶段三 【 Web 相关领域的基础 - HTML / CSS / JavaScript 】

### 阶段时长：`10` 天

### 🛒学习内容

❗❗❗**注意：本阶段的各个学习内容存在强联动，不建议单独学习，建议分别进行基础语法了解后联动学习。重点学习 HTML 与 CSS 。**

1. 学习 HTML 与 CSS 基础，了解 JS，并综合应用
2. 学会用 `GitHub Pages` 部署静态页面

### 📑任务目标

1. 👉使用 HTML、CSS 与 JS 制作**一个属于你自己的静态网页**，展示任何你想展示的内容，然后使用`Github Pages` 部署该页面。👈

- **致如此努力的你**： 

  坚持到现在的你非常棒！
  而本项目将是你第一个完整实现的前端项目，或许有许多 BUG、障碍挡路，
  但我们希望你用心思考，多多实践、终能披荆斩棘，用一行行代码展示出你最耀眼的光辉。

- **我们鼓励创新，但请力所能及**：

  如学有余力，可以添加独属于你的创新亮点，
  但我们不希望直接借助 AI 实现本不属于你的东西，
  也不希望你为了创新而忽视了基本。

- **我们允许简单，只要你尽力**：

  如果你确实时间较少或者学习相对吃力，
  可以只用 HTML、CSS 与 JS 的一两种制作一个简单的页面demo，
  尽力而为并持续学习，一定会成为优秀的人。

2. 无论你的项目如何，详细编写一篇 “编后感” ，说说：
   - 你构想了一个什么项目，
     最终完全实现你的构想了吗？
   - 你在项目中，是如何用 HTML 给它搭建骨骼、用 CSS 给它增添颜值、用JS给它赋予变化的？
   - 我们总是在关关难过关关过中成长，
     说说你在项目中遇到的最大难题是啥？你又是如何克服和解决的？
   - 你认为当前的项目还需要优化和补完吗？
     如果将来你成为一方 dalao，你会想回到最初为这个项目填上最后一块砖吗？
